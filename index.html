授業を重ねるごとに基礎知識を増やしていき応用していきより実践的な能力を身につけることが目的。 目標はHTML等をうまく使いこなし自分一人の力でWebページの作成ができるようにする

HTMLを書く上では1回目の簡単な内容を使用することで作成することができた。ulとliをうまく使うことがWebサイトを作る時重要なことだな感じることができた。回を重ねるごとにやることが増えて、最終回は今までの内容が全部出てきた感じだった。テキストボックスを作ったり、ajaxで通信したりするのは難しかったけど、少しずつ理解できるようになってきて、自分で動かせたときは嬉しかった。

数当てゲーム

1〜9までの数字を予想していきランダムで指定された数字を当てるゲームです。また1回間違えるごとにヒントが出てくるのでそのヒントを頼りに数字を当ててください。inputやbuttonの要素を追加したことで、見た目にも操作にもゲームっぽさが出てきて、遊んでる感じが強くなった。ユーザーが実際に入力したりボタンを押したりできるから、よりリアルな動きが楽しめる。ただし、addEventListenerを使うときに、どの処理を動かすかをしっかり指定しておかないと、思ったように動かなくて困ることがある。イベントの設定を適当にやってしまうと、うまく反応しなかったりバグの原因になることもあるから、そこはちゃんと意識してコードを書いたほうがいいなと感じた。

各授業回の学習事項

第一回WWWとHTML

HTMLで見出し、箇条書き、番号付きリスト、リンク、定義リスト、画像の挿入方法を学び、情報を整理して分かりやすく構成する手法を学んだ。

演習の成果：演習1-4:HTMLのさまざまな要素

第二回cssによるWebページの装飾

cssは主にセレクタ、プロパティ名、プロパティと三つに分かれていることがわかった。webをうまく作るにはフレックスボックス、グリットレイアウトを使うと見やすいwebが作れると思った。

演習の成果：演習2-3:cssによるWebページの装飾

第三回JavaScriptの基本

1回目の授業では段落や文字を大きく表示するやり方画像を貼り付けるやり方などを学んだ。 2回目の授業では色をつけたり色を変えたり画像の位置をずらしたりするやり方を学んだ。 第３回では数当てゲームといった簡単なゲームが作れるようになった。javaと似ている部分が多いけど違うところもあるので、ごっちゃにならないように気をつける。

演習の成果：演習3-6:第三回JavaScriptの基本

第四回DOMとJava Script

DOMを使うと、すべてのメンバー変数を検索・変更・削除できる。また、CSSのプロパティを設定するときは、マイナス記号を使うものと使わないものがあるので、記述方法に注意が必要だ。

演習の成果：演習4-10:第四回演習の成果

第五回イベント処理

イベント処理のやり方を学んで、どこでイベントが設定されているかを確認するには、インスペクターで「event」と出ているかを見ればよいとわかった。これをすることでミスを防ぎやすくなる。テキストボックスを使うときは、valueを使ってちゃんと中の文字を読み込まないと、出てくる値が変になってしまうので注意が必要。

演習の成果：演習５-7:第五回演習の成果

感想

情報メディアの授業を通して、最初に思っていたよりもずっと深く、HTML、CSS、JavaScript、そしてDOMについて学ぶことができた。Webページの構造や見た目を作る仕組みがよくわかるようになり、とてもおもしろかった。 HTMLでは、要素や属性の役割をしっかり理解できたほか、特殊文字の使い方も出てきて、記号の意味が難しかったのが印象に残っている。MDN Web Docsというサイトを使って新しいタグを調べたり、さまざまな表現方法を見つけたりできたことで、課題や授業の演習にとても役立った。HTMLは今回の授業の中で一番好きな内容だった。 CSSでは、Webサイトの見た目を整える作業が楽しく、課題も楽しみながら取り組めた。たとえば、練習2-1で文字の色がなかなか変わらず戸惑ったけれど、ミスを見つけて修正できた時は嬉しかった。CSSはプロパティ名と値を細かく調整できる点が魅力だと感じた。また授業が進む中で、より専門的なツールも登場し、本格的にWeb開発するのは想像以上に大変だとも感じた。 JavaScriptの授業では、まずコンソールに文字を表示するところから始まり、一見簡単そうだったが、覚えることが多くて、どこでどのコードを使えばいいのか迷う場面も多かった。Javaと似ている点もあって比較はしやすかったが、自分にとっては一番難しく、苦手に感じた内容だった。 DOMに入ってからは急に難易度が上がった印象があり、課題や演習についていくのが少し大変になった。ただ、DOMを使えば、指定した要素に直接命令を与えることができて、とても便利な機能だと感じた。 イベント処理では、テキストボックスやボタンを作成し、クリックやマウス操作などに応じて動作するプログラムを学んだ。マウスが通過したり、クリックしたりしたときに反応するイベントの仕組みが面白かった。イベントが正しく設定されているかはインスペクターで確認できるのも便利だった。DOMとイベント処理が組み合わさったときに、「Webが動いている」感じがしてとてもワクワクした。 中でも一番大変だったのはProject課題だった。何度もエラーが出て、どこが間違っているのかわからず、試行錯誤の連続だった。特にURLからデータを読み込んで、そこから値を取り出して表示する部分は難易度が高かった。けれど、最終的に自分でWebページを一から作りあげる経験ができて、本当にいい学びになったし、達成感も大きかった。
